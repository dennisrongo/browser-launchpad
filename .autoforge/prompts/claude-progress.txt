### Technical Implementation Notes

**Storage Architecture:**
- All persistent data flows through `src/services/storage.ts`
- No in-memory data stores (no Maps, Sets, or globalThis patterns)
- All React state mirrors Chrome storage, not replaces it
- Storage writes are verified by reading back after write
- Storage changes are monitored via chrome.storage.onChanged

**Chrome Storage API Usage:**
- Permission: "storage" in manifest.json ✓
- API: chrome.storage.local (not chrome.storage.sync)
- All operations are async with Promise wrappers
- Error handling for chrome.runtime.lastError
- Console logging for debugging (visible in DevTools)
